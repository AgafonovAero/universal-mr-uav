# Системы координат и соглашения

## Базовые соглашения

- земная система координат: `NED`
- связанная система координат: `X` вперед, `Y` вправо, `Z` вниз
- внутренняя ориентация: кватернион
- углы Эйлера допускаются только для отображения, отладки и отдельных интерфейсов
- единицы измерения: только `SI`
- углы в коде: только радианы

## Состояние модели

Жесткое тело:

`x_rigid = [p_ned_m; v_b_mps; q_nb; w_b_rps]`

Минимальное расчетное ядро `Stage-1.5`:

`x_plant = [p_ned_m(3); v_b_mps(3); q_nb(4); w_b_rps(3); omega_m_radps(4)]`

где:

- `p_ned_m` - положение в `NED`, м
- `v_b_mps` - линейная скорость в связанной системе координат, м/с
- `q_nb` - scalar-first кватернион поворота из body в `NED`
- `w_b_rps` - угловая скорость в связанной системе координат, рад/с
- `omega_m_radps` - угловые скорости четырех роторов, рад/с

## Силы, моменты и геометрия

- положительная тяга ротора направлена вдоль `-Z_b`
- суммарная сила тяги записывается как `forces_b_N = [0; 0; -sum(T_i)]`
- моменты от роторов считаются по формулам:
  - `Mx = sum(-y_i * T_i)`
  - `My = sum( x_i * T_i)`
  - `Mz = sum(spin_dir_i * Q_i)`

Геометрия `quad-X` задается через:

- `wheelbase_m` - диагональ между противоположными роторами
- `motor_radius_m` - расстояние от центра до ротора
- `motor_xy_m` - координаты роторов в виде `[x_i, y_i]`

Для симметричного `quad-X`:

- `motor_radius_m = wheelbase_m / 2`
- `abs(x_i) = abs(y_i) = motor_radius_m / sqrt(2)`

## Команды двигателя

Команды двигателя задаются напрямую как целевые скорости роторов:

`motor_cmd_radps = [omega_1; omega_2; omega_3; omega_4]`

Сначала эти команды проходят через модель `ESC + motor`, затем преобразуются в силы и моменты, и только после этого подаются в модель жесткого тела.

## Допущения Stage-1 / Stage-1.5

- объект управления рассматривается как твердое тело
- масса и тензор инерции постоянны
- среда включает только гравитацию
- модель ВМГ квадратична по скорости ротора
- `quad-X` остается единственной схемой смешения
- Simulink shell на этом этапе отсутствует, источник истины - `.m` код и текстовые файлы
