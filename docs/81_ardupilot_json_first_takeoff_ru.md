# Первый взлетный прогон ArduPilot JSON на математической модели БВС

## 1. Назначение документа

Документ фиксирует результаты TASK-23 по первому управляемому взлетному прогону в режиме
`ArduPilot JSON + MATLAB-модель` после того, как в TASK-22 была получена принятая команда
взведения.

Здесь рассматривается именно стендовая инженерная диагностика причинно-следственной цепи:

`команда ArduPilot -> ШИМ -> частота вращения винтов -> тяга -> изменение состояния модели`.

Документ не является отчетом о летной валидации и не подтверждает устойчивый автоматический
полет.

## 2. Исходное состояние перед TASK-23

После TASK-22 было подтверждено:

- предполетный блокер по частоте гироскопа снят профилем
  `tools/ardupilot/wsl/task22_sched_loop_200.parm`;
- `arm = да`;
- `COMMAND_ACK = 0`;
- диапазон `motor_pwm_us = 1000..1100 мкс`;
- диапазон `motor_cmd_radps = 0..90 рад/с`;
- устойчивый `JSON/UDP`-обмен сохранен.

Следовательно, первый взлетный прогон должен был ответить уже не на вопрос
"можно ли взвести ArduPilot", а на вопрос "дает ли этот взведенный режим тягу,
достаточную для роста высоты математической модели".

## 3. Что показали параметры моторных выходов

Сценарий `scripts/diagnose_ardupilot_motor_output_params.m` подтвердил параметры группы
движителей ArduPilot:

- `MOT_PWM_MIN = 1000`;
- `MOT_PWM_MAX = 2000`;
- `MOT_SPIN_ARM = 0.10`;
- `MOT_SPIN_MIN = 0.15`;
- `MOT_SPIN_MAX = 0.95`;
- `MOT_THST_HOVER = 0.39`;
- `MOT_THST_EXPO = 0.65`.

Это означает, что ArduPilot настроен на обычную шкалу ШИМ `1000..2000 мкс`, однако его
текущая оценка уровня зависания (`MOT_THST_HOVER`) соответствует примерно `1390 мкс`,
что ниже фактического уровня зависания модели.

## 4. Что показал расчет тягового баланса

Сценарий `scripts/diagnose_pwm_to_thrust_balance.m` связал выходной ШИМ ArduPilot с
моделью винтомоторной группы.

Ключевой результат:

- расчетный уровень зависания модели находится около `1600 мкс`;
- при `1100 мкс` суммарная тяга составляет лишь `2.64 %` веса;
- при `1600 мкс` суммарная тяга почти равна весу (`0.9512`);
- при `1650 мкс` суммарная тяга уже превышает вес;
- при `2000 мкс` модель имеет запас тяги более `2.64` веса.

Следовательно:

- сама математическая модель допускает отрыв и набор высоты;
- проблема TASK-23 заключается не в невозможности взлета как таковой, а в том,
  что ArduPilot в рассматриваемых опытах не выдал команду ШИМ, близкую к
  расчетному зависанию;
- простое наличие `arm` еще не означает готовность к набору высоты.

## 5. Результат опыта arm без takeoff

Сценарий `scripts/run_ardupilot_json_arm_idle_observation.m` показал:

- `arm` принимается;
- каналы ШИМ действительно выходят из строго минимального уровня и достигают
  `1000..1100 мкс`;
- команды частоты вращения доходят до `0..90 рад/с`;
- максимальное отношение тяги к весу составляет только `0.0264`.

Это важный промежуточный результат:

- цепочка `arm -> PWM -> частота вращения -> тяга` подтверждена;
- но величина тяги после `arm` недостаточна для вертикального отрыва.

Рост высоты вверх в этом опыте не подтвержден.

## 6. Результат GUIDED takeoff 1 м

Сценарий `scripts/run_ardupilot_json_guided_takeoff_1m.m` показал:

- режим `GUIDED` наблюдается;
- команда `arm` не принимается;
- первая причина отказа:
  `Arm: GPS 1: Bad fix`;
- команда `takeoff` не принимается;
- ШИМ и команды винтов остаются на нуле относительно тяги
  (`motor_pwm_us = 1000..1000 мкс`,
  `motor_cmd_radps = 0..0 рад/с`).

То есть первый взлетный прогон уперся не в недостаток тяги после принятого `takeoff`,
а в то, что автоматический режим `GUIDED` вообще не разрешил перейти к выдаче тяги
из-за предполетного блокера по GPS.

## 7. Результат throttle step

Дополнительный сценарий `scripts/run_ardupilot_json_throttle_step_response.m` был нужен,
чтобы проверить более прямое воздействие на канал тяги, если `GUIDED takeoff` не срабатывает.

В последнем актуальном прогоне получено:

- `arm = нет`;
- причина отказа:
  `Arm: Gyro 0 rate 348Hz < loop rate*1.8 360Hz`;
- роста ШИМ и команд винтов не наблюдается.

Следовательно, этот диагностический путь пока тоже не дал роста тяги, но уже по другой
причине: возвратился блокер предполетной проверки по частоте гироскопа.

## 8. Диагностический профиль тяги

Для проверки гипотезы о слишком низком уровне тяги после `arm` был подготовлен
диагностический профиль:

- `tools/ardupilot/wsl/task23_hover_thrust_diagnostic.parm`

с повышением `MOT_THST_HOVER`, `MOT_SPIN_ARM` и `MOT_SPIN_MIN`.

Он не принят как штатное решение, поскольку:

- вводился только как диагностический вариант;
- не улучшил прохождение команды `arm`;
- не подтвердил рост тяги до уровня, достаточного для взлета.

## 9. Инженерный вывод по TASK-23

TASK-23 дал не летный, а диагностический результат:

- подтверждено, что после `arm` ArduPilot способен сформировать ненулевой ШИМ и ненулевые
  команды частоты вращения винтов;
- подтверждено, что уровень `1000..1100 мкс` недостаточен для отрыва математической модели;
- автоматический `GUIDED takeoff 1 м` упирается в `GPS 1: Bad fix`;
- дополнительный опыт с диагностической командой газа в последнем прогоне упирается в
  возврат блокера по частоте гироскопа.

Поэтому допустимо утверждать только следующее:

- первичная реакция стенда на команду взведения подтверждена;
- первый взлетный прогон в режиме `ArduPilot JSON + MATLAB-модель` выполнен;
- рост высоты вверх пока не подтвержден.

## 10. Что не является летной валидацией

Результаты TASK-23 не являются:

- летной валидацией;
- подтверждением устойчивого автоматического взлета;
- подтверждением корректной работы всех режимов ArduPilot на внешней математической модели;
- подтверждением правильности траекторного управления.

## 11. Следующий этап

Следующий этап должен быть направлен на устранение именно тех причин, которые не дали
получить рост тяги и высоты в TASK-23:

- обеспечить приемлемое состояние навигации для `GUIDED`-режима либо доказать, что
  для стенда нужен другой штатный путь выдачи команды тяги;
- стабилизировать диагностический путь ручной команды газа без возврата блокера
  по частоте гироскопа;
- после снятия этих ограничений повторить опыт `GUIDED takeoff` и проверить
  знак вертикального движения, а также баланс тяги и веса без заявлений
  о летной валидации.
