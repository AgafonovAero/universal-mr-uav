# Plant API Stage-1.5

## Каноническая структура состояния

Внешний API расчетного ядра использует struct состояния со строго заданными полями:

- `p_ned_m` - положение в земной системе координат `NED`, размер `3x1`
- `v_b_mps` - линейная скорость в связанной системе координат, размер `3x1`
- `q_nb` - scalar-first кватернион поворота из body в `NED`, размер `4x1`
- `w_b_radps` - угловая скорость в связанной системе координат, размер `3x1`
- `omega_m_radps` - угловые скорости четырех роторов, размер `4x1`

Проверка канонической структуры выполняется через `uav.core.state_validate`.

## Packed-вектор состояния

Для совместимости с существующим Stage-1.5 kernel используется packed-вектор порядка:

```text
[p_ned; v_b; q_nb; w_b; omega_m]
```

Размер packed-вектора: `17x1`.

Преобразования выполняются функциями:

- `uav.core.state_pack(state)`
- `uav.core.state_unpack(x_packed)`

## Сигнатура plant_step_struct

```matlab
[state_next, diag] = uav.sim.plant_step_struct(state_prev, motor_cmd_radps, dt_s, params)
```

Где:

- `state_prev` - канонический struct состояния
- `motor_cmd_radps` - вектор `4x1` ссылок по угловой скорости роторов
- `dt_s` - шаг интегрирования
- `params` - параметрический preset объекта управления и ВМГ

`plant_step_struct` внутри использует существующий `uav.sim.plant_step`.

### Состав diag

`diag` повторяет диагностический формат нижнего уровня:

- `motor` - диагностика дискретной модели `ESC + motor`
- `fm` - агрегированные силы и моменты от ВМГ
- `omega_m_radps` - скорость роторов после шага
- `forces_b_N` - суммарная сила в связанной системе координат
- `moments_b_Nm` - суммарный момент в связанной системе координат
- `quat_norm` - норма кватерниона после шага
- `dx_rigid` - производная 13-мерного rigid-body состояния на шаге

## Сигнатура run_case

```matlab
log = uav.sim.run_case(case_cfg)
```

Обязательные поля `case_cfg`:

- `params`
- `state0`
- `dt_s`
- `t_final_s`
- `command_fun`

### Формат command_fun

```matlab
motor_cmd_radps = command_fun(t_s, state, params)
```

Где:

- `t_s` - текущее модельное время
- `state` - текущее каноническое состояние
- `params` - preset параметров

Результат должен быть вектором `4x1` в `rad/s`.

## Формат log

`run_case` возвращает прозрачный log без скрытого состояния:

- `time_s` - сетка времени `Nx1`
- `state` - массив `Nx1` канонических struct-состояний
- `motor_cmd_radps` - история команд `Nx4`
- `omega_m_radps` - история скоростей роторов `Nx4`
- `forces_b_N` - история суммарных сил `Nx3`
- `moments_b_Nm` - история суммарных моментов `Nx3`
- `quat_norm` - история нормы кватерниона `Nx1`

## Системы координат и знаковые соглашения

- земная система координат: `NED`
- связанная система координат: `X forward`, `Y right`, `Z down`
- внутренняя ориентация: quaternion
- положительная тяга ротора действует вдоль `-Z_b`
- положительный yaw moment задается вокруг оси `+Z_b`

Эти соглашения не меняются в рамках TASK-04-RU.
