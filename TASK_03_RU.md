- Сохранять обратную совместимость там, где это не мешает задаче.

## Что нужно сделать

### 1. Уточнить baseline preset квадрокоптера
Обновить:

Требуется:
- сохранить существующие поля, если они уже используются тестами;
- дополнительно ввести явные параметры ВМГ:

- допускается дублирование legacy-полей и новых вложенных полей ради совместимости Stage-1.

### 2. Добавить модель ESC+двигателя
Создать:
- `src/+uav/+vmg/motor_esc_step.m`

Функция должна реализовывать простую дискретную модель первого порядка:
- вход:
  - `omega_prev_radps` (4x1)
  - `omega_cmd_radps` (4x1)
  - `dt_s`
  - `motor_params`
- выход:
  - `omega_next_radps` (4x1)
  - `diag` struct

Требования:
- saturation по `omega_min_radps`, `omega_max_radps`;
- апериодическое звено первого порядка:
  - `domega = (omega_cmd_sat - omega_prev) / tau_s`
  - `omega_next = omega_prev + dt_s * domega`
- повторная saturation после шага;
- никаких скрытых параметров внутри функции.

### 3. Добавить агрегатор сил и моментов
Создать:
- `src/+uav/+core/forces_moments_sum.m`

Назначение:
по текущим скоростям роторов вычислять суммарные силы и моменты в связанной системе координат.

Вход:
- `omega_radps` (4x1)
- `params`

Выход:
- `fm` struct со следующими полями:
  - `rotor_thrust_N`
  - `rotor_torque_Nm`
  - `forces_b_N`
  - `moments_b_Nm`

Требования:
- использовать существующий `uav.vmg.rotor_simple`;
- считать, что положительная тяга ротора направлена вдоль `-Z_b`;
- использовать `params.motor_xy_m` и `params.spin_dir`;
- формулы моментов:
  - `Mx = sum(-y_i * T_i)`
  - `My = sum( x_i * T_i)`
  - `Mz = sum(spin_dir_i * Q_i)`
- суммарная сила:
  - `forces_b_N = [0; 0; -sum(T_i)]`

### 4. Добавить минимальный plant step
Создать:
- `src/+uav/+sim/plant_step.m`

Назначение:
выполнить один дискретный шаг моделирования объекта управления.

Состояние:
- использовать вектор состояния длины 17:
  - `[p_ned(3); v_b(3); q_nb(4); w_b(3); omega_m(4)]`

Вход:
- `x_prev`
- `motor_cmd_radps`
- `dt_s`
- `params`

Выход:
- `x_next`
- `diag`

Требования:
- обновление состояний двигателя делать через `uav.vmg.motor_esc_step`;
- силы и моменты считать через `uav.core.forces_moments_sum`;
- для жесткого тела использовать существующий `uav.core.eom6dof_quat`;
- интегрирование жесткого тела сделать простым и прозрачным:
  - допускается explicit Euler или RK4;
- после шага обязательно нормализовать кватернион;
- в `diag` вернуть:
  - текущие скорости роторов,
  - суммарную силу,
  - суммарные моменты,
  - норму кватерниона.

### 5. Добавить демонстрационные сценарии
Создать:
- `scripts/run_motor_spool_demo.m`
- `scripts/run_openloop_hover_demo.m`

`run_motor_spool_demo.m`:
- моделирует раскрутку роторов из нуля до постоянной команды;
- выводит в консоль диагностические величины;
- допускает построение простых графиков.

`run_openloop_hover_demo.m`:
- моделирует открытый контур при постоянной команде на hover;
- использует `plant_step`;
- выводит диагностические величины:
  - hover command
  - final rotor speeds
  - total thrust
  - weight
  - vertical acceleration estimate
