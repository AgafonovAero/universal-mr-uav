# TASK-10: средство сопряжения с ArduPilot для программного моделирования

## Назначение задачи

TASK-10 предназначена для подготовки средства сопряжения между
`universal-mr-uav` и внешним комплексом управления полетом `ArduPilot`
через будущий JSON-обмен данными.

На текущем этапе задача решает подготовительную инженерную цель:

- зафиксировать состав данных для будущего обмена;
- определить типовую конфигурацию каналов связи;
- задать преобразование команд ШИМ в команды частоты вращения винтов;
- провести проверочные замкнутые прогоны без реального внешнего
  автопилота;
- оформить результаты в виде воспроизводимых документов и журналов.

Задача не ставила целью:

- полноценное сопряжение с работающим `ArduPilot SITL`;
- подтверждение готовности к штатной эксплуатации;
- выполнение летной валидации.

## Выполненные изменения

В составе TASK-10 создан пакет `src/+uav/+ardupilot/`.

В него включены:

- функция типовой конфигурации JSON-обмена;
- функция проверки локальной готовности вычислительной среды;
- функция формирования канонического пакета данных состояния и
  измерений;
- функция разбора команд широтно-импульсной модуляции;
- функция преобразования ШИМ в команды частоты вращения винтов;
- функция структурной проверки пакета данных.

Кроме того, в задаче подготовлены:

- функция формирования тестового пакета команд;
- исполнитель сценария моделирования для проверочных замкнутых прогонов;
- сценарий проверки локальной среды;
- сценарий режима зависания;
- сценарий ступенчатого воздействия по рысканию;
- комплект модульных проверок;
- документация по средству сопряжения;
- исходные журналы прогонов.

## Готовность локальной среды

Проверка локальной среды дала следующие результаты:

- `ready for real SITL = no`;
- `has WSL = no`;
- `has Python = yes`;
- `has sim_vehicle.py = no`.

Также локальный каталог `ArduPilot` не обнаружен.

Это означает, что локальная среда пока не готова к следующему этапу
реального сопряжения. Для перехода к нему как минимум отсутствуют:

- `WSL`;
- рабочий checkout `ArduPilot`;
- сценарий запуска `sim_vehicle.py`.

## Ограничения текущего этапа

На текущем этапе отсутствуют:

- реальный обмен данными по UDP;
- работающий экземпляр `ArduPilot SITL`;
- согласование всех режимов и порядка двигателей;
- подтверждение устойчивого замкнутого полета с внешним автопилотом;
- летная валидация.

Следовательно, результаты TASK-10 не должны трактоваться как завершенное
сопряжение с `ArduPilot`. Это только инженерная подготовка интерфейса и
проверка внутренней согласованности расчетного ядра с новой логикой
обмена.
