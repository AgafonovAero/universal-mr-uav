## Фактический запуск ArduPilot SITL

Для подтверждения запуска использовались два способа.

### Запуск через существующий однокнопочный сценарий

Был выполнен режим:

```matlab

cfg = struct( ...
    'execute_start', true, ...
    'launch_mission_planner', false, ...
    'launch_qgroundcontrol', false);
OneKeyArduPilotStand("start", cfg)
```

По журналу `artifacts/logs/task_15_start_ardupilot_json_sitl.txt`
подтверждено:

- сценарий `tools/ardupilot/windows/Start-ArduPilotStand.ps1`
  был запущен в режиме выполнения;
- внутри `WSL` был выдан фактический запуск `ArduPilot SITL`;
- в рамках этого запуска был принят реальный двоичный пакет;
- после принятого пакета была выполнена ответная передача строки `JSON`.

### Устойчивый фоновый запуск для повторных проверок

Для последующих отдельных прогонов применялся запуск в скрытом окне
`PowerShell`, который удерживал процесс `arducopter` в фоне:

```powershell
wsl -d Ubuntu -- bash -lc 'cd ~/src/ardupilot/ArduCopter && source ~/.profile >/dev/null 2>&1 && exec ../build/sitl/bin/arducopter --model JSON:172.19.208.1 --speedup 1 --slave 0 --serial0=udpclient:172.19.208.1:14550 --defaults ../Tools/autotest/default_params/copter.parm --sim-address=172.19.208.1 -I0'
```

Факт существования процесса подтверждался через:

```powershell
wsl -d Ubuntu -- bash -lc 'pgrep -af arducopter || true'
```

Типовой подтвержденный экземпляр процесса:

- `396 ../build/sitl/bin/arducopter --model JSON:172.19.208.1 ...`

Следовательно, факт запуска `ArduPilot SITL` подтвержден.
