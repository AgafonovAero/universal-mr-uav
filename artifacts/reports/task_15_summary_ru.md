# TASK-15. Фактическая попытка установки и запуска стенда ArduPilot

## Назначение задачи

TASK-15 предназначен для перехода от безопасных проверочных прогонов
TASK-14 к фактическому выполнению уже созданного однокнопочного стенда
`ArduPilot SITL` с математической моделью `universal-mr-uav`.

На данном этапе не ставилась задача переработки архитектуры стенда,
изменения математической модели движения, изменения алгоритмов
оценивания состояния, изменения коэффициентов или изменения моделей
`Simulink`.

Целью являлась именно фактическая проверка:

- установлен ли дистрибутив `Ubuntu` внутри `WSL`;
- можно ли реально подготовить внешний каталог `ArduPilot`;
- можно ли перейти к запуску `ArduPilot SITL` в режиме `JSON/UDP`;
- можно ли получить первый двоичный пакет от `ArduPilot`;
- можно ли перейти к ответной передаче строки `JSON`.

## Выполненные действия

Локально были фактически выполнены:

```matlab
run('scripts/bootstrap_project.m')
OneKeyArduPilotStand("check")
OneKeyArduPilotStand("status")
run('scripts/bootstrap_project.m')
runtests('tests')
```

Также была выполнена фактическая попытка реальной подготовки `Ubuntu`
внутри `WSL` через уже существующий PowerShell-сценарий:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\ardupilot\windows\Setup-WSLForArduPilot.ps1 -DistroName Ubuntu -Execute
```

## Результат фактической попытки установки Ubuntu в WSL

По журналу `artifacts/logs/task_15_setup_ubuntu_wsl.txt` подтверждено,
что сценарий `Setup-WSLForArduPilot.ps1` был вызван в режиме
`-Execute`, однако выполнение остановилось на первом конкретном
ограничении:

- текущий сеанс PowerShell не обладает правами администратора;
- сценарий корректно определил, что для установки `Ubuntu` требуются
  повышенные права;
- сценарий не выполнил скрытых действий и не попытался продолжать
  установку без необходимых прав.

Фактически подтверждено:

- `Ubuntu` не была установлена;
- `WSL` не был административно изменен;
- дальнейшая подготовка `ArduPilot` внутри `WSL` не выполнялась.

## Первая конкретная причина остановки

Первая конкретная причина остановки выполнения:

**отсутствие повышенных прав PowerShell для установки дистрибутива
`Ubuntu` в `WSL`.**

Сценарий выдал точную рекомендуемую команду для оператора.

Оператор должен открыть PowerShell **от имени администратора** и
выполнить:

```powershell
cd D:\WORK\AI_SOLUTION\universal-mr-uav
powershell -ExecutionPolicy Bypass -File .\tools\ardupilot\windows\Setup-WSLForArduPilot.ps1 -DistroName Ubuntu -Execute
```

До выполнения этой команды переход к следующим этапам невозможен.

- установка `Mission Planner`;

отсутствии `Ubuntu`, прав администратора, `sudo` или другой критической
зависимости выполнение не должно продолжаться искусственно.
